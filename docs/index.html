Instalação

Instalador Web (Recomendado)

Acesse /install após fazer o upload dos arquivos. O instalador guiará pelos passos de verificação de requisitos, configuração do banco de dados, dados do sistema e criação do administrador.

Instalação Manual

  1. Clone o repositório e configure o ambiente:
    git clone <repo> hostpanel
cp .env.example .env
composer install --no-dev --optimize-autoloader
  2. Configure o arquivo .env com as credenciais do banco de dados e outras configurações.
  3. Gere a chave da aplicação e execute as migrations:
    php artisan key:generate
php artisan migrate --seed
php artisan storage:link
  4. Instale as dependências frontend e compile:
    npm install
npm run build
  5. Configure o supervisor para as filas:
    [program:hostpanel-worker]
command=php artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
directory=/var/www/hostpanel
autostart=true
autorestart=true
numprocs=4
  6. Configure o cron para tarefas agendadas:
    * * * * * cd /var/www/hostpanel && php artisan schedule:run >> /dev/null 2>&1
ℹ️
Após a instalação manual, marque a aplicação como instalada criando o arquivo storage/installed ou definindo INSTALLED=true no .env.

API REST v1

URL Base: https://seudominio.com/api/v1

Autenticação: Bearer Token via header Authorization: Bearer {token}

⚠️
Todos os endpoints retornam JSON. Tokens são gerados via POST /api/v1/auth/login e têm expiração configurável.

API — Autenticação

POST /api/v1/auth/login Autenticar e obter token

Body:

{ "email": "cliente@email.com", "password": "senha" }

Resposta 200:

{ "token": "hp_...", "token_type": "Bearer", "client": { ... } }
GET /api/v1/auth/me Dados do cliente autenticado

Requer header Authorization: Bearer {token}

POST /api/v1/auth/logout Revogar token atual

Invalida o token Bearer usado na requisição.

API — Clientes

GET /api/v1/clients Listar clientes (paginado)

Query params: search, status, page, per_page

GET /api/v1/clients/{id} Detalhes do cliente + serviços e faturas

Inclui relacionamentos services e invoices.

POST /api/v1/clients Criar cliente 
{ "name": "João Silva", "email": "joao@email.com", "password": "senha123" }

API — Serviços

GET /api/v1/services Listar serviços

Query params: client_id, status, page

POST /api/v1/services/{id}/suspend Suspender serviço 
{ "reason": "Não pagamento" }
POST /api/v1/services/{id}/reactivate Reativar serviço

Chama o ProvisioningService::reactivate() via WHM API.

API — Faturas

Query params: client_id, status, page

Provisionamento WHM/cPanel

O WHMService encapsula toda comunicação com o servidor WHM via API v1:

// Criar conta cPanel
$whm = new WHMService($server);
$result = $whm->createAccount([
    'username'  => $service->username,
    'domain'    => $service->domain,
    'plan'      => $product->cpanel_plan,
    'password'  => $service->password,
]);

// Gerar link SSO para cPanel
$url = $whm->createAutoLoginUrl($service->username);

Fluxo de Provisionamento

  1. Cliente realiza pedido e pagamento é confirmado.
  2. InvoicePaidEvent é disparado.
  3. ProvisionServicesListener coloca ProvisionServiceJob na fila.
  4. Worker processa o job e chama WHMService::createAccount().
  5. Serviço é atualizado para active e cliente é notificado por e-mail.

Módulo Financeiro

Gerenciado pelo BillingService. Suporta ciclos: monthly, quarterly, semiannually, annually, biennially, triennially, free.

Fluxo de Pagamento

  1. Fatura criada com status pending e data de vencimento.
  2. Cliente acessa a fatura e seleciona o gateway de pagamento.
  3. Gateway processa o pagamento (PIX, boleto, cartão, etc.).
  4. Webhook do gateway notifica o sistema → GatewayController::webhook().
  5. BillingService::applyPayment() registra a transação e atualiza a fatura para paid.
  6. InvoicePaidEvent dispara provisionamento e notificações.

Regras de Cobrança

  • Faturas geradas N dias antes do vencimento (configurável)
  • Multa + juros automáticos após X dias de atraso
  • Suspensão automática após Y dias sem pagamento
  • Saldo de crédito pode ser usado como abatimento parcial ou total
  • Cupons: percentual ou valor fixo, por cliente ou uso único

PWA — Progressive Web App

O HostPanel inclui suporte completo a PWA com:

  • Service Worker com cache inteligente (network-first HTML, cache-first assets)
  • Manifest JSON gerado dinamicamente via PwaController
  • Página offline automática quando sem conexão
  • Push Notifications via Web Push API
  • Instalável em iOS, Android e Desktop
✅
O Service Worker é servido via rota Blade (/service-worker.js) para ter acesso às variáveis do Laravel, mas com header Content-Type: application/javascript.

Segurança

  • CSRF Protection em todas as rotas web (exceção: webhooks de gateways)
  • Rate Limiting no login (5 tentativas/min) e na API
  • API Tokens hashados com SHA-256 no banco de dados
  • Senhas hashadas com bcrypt (custo 12)
  • SQL Injection — prevenida por Query Builder e Eloquent ORM
  • XSS — prevenido por escaping automático do Blade
  • Impersonação auditada com IP, admin e motivo em log dedicado
  • 2FA TOTP disponível para admins e clientes
  • HTTPS obrigatório em produção (configurar no servidor web)
  • Headers de segurança (HSTS, X-Frame-Options, CSP) recomendados via Nginx/Apache
